Detailed Description

This header file declares the GameManager.

The GameManager is responsible for keeping track of the state of the game and mediating communication between the different managers and GameObjects/Components of the game. This is what ensures low coupling in my application as every GameObject every only has one object to talk to and the GameManager also mediates all information from Managers. One might say that this gives GameManager High Coupling and Low Cohesion, but I disagree. It is only doing one thing, and it does it well.
